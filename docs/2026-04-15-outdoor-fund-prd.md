## 1. 产品概述

### 1.1 产品定位

面向户外活动群组的**公积金自动结算系统**。成员预充值公积金余额，每次活动由系统自动 AA 计算并扣减余额，余额不足时自动转为现金补款，替代手工记账和线下收款。

### 1.2 核心价值

| 痛点 | 解决方案 |
|------|---------|
| 每次活动手工算 AA，容易出错 | 系统自动结算，支持外挂、按车分摊等复杂场景 |
| 垫付后催款困难 | 预充值余额自动扣减，余额不足标记补款 |
| 多个群各管各账，数据混乱 | 多群隔离，余额独立，统一平台管理 |
| 活动结算后发现错误无法修改 | 冲红重算机制，修改自动回滚+重新结算 |
| 资金流水不透明 | 不可变账本 + 审计日志，每笔变动可追溯 |

### 1.3 目标用户

户外活动群组（登山、徒步、自驾、露营等），群规模 5~100 人。

### 1.4 产品形态

移动端优先的 H5 Web 应用，部署在阿里云 ECS，通过手机浏览器访问。

## 3. 功能模块

### 3.1 群组管理

**目的**：支持同一用户管理多个独立的活动群。

- **创建群组**：输入群名称（如"周末登山群"）、管理员密码、会员密码、群简介。
- **群组列表**：展示所有群组名称，点击进入。
- **群组切换**：登录后可切换到其他已加入的群组。
- **数据隔离**：A 群的充值仅限 A 群消费，余额、活动、会员列表均独立。
- **成员互通**：同一个人可加入多个群组，但在每个群中有独立余额。

### 3.2 会员管理

**目的**：管理群组成员信息和公积金余额。

#### 3.2.1 会员列表

| 展示字段 | 说明 |
|---------|------|
| 姓名 | 会员名称 |
| 当前余额 | **醒目展示**（正数绿色，负数红色） |
| 累计活动次数 | 参加活动的总次数 |

- 支持按余额排序、按姓名搜索。
- 点击进入会员详情页。

#### 3.2.2 会员操作

| 操作 | 说明 |
|------|------|
| 新增会员 | 输入姓名（必填）、手机号（选填）、头像（选填） |
| 编辑会员 | 修改基本信息 |
| 手动充值 | 输入金额 → 余额增加，生成 `topup` 类型流水记录 |
| 一键收款充值 | 批量选择欠款会员 → 统一输入金额 → 批量充值并扣费 |

#### 3.2.3 个人流水

按时间倒序展示该会员在当前群的全部资金变动记录：

| 字段 | 说明 |
|------|------|
| 时间 | 操作发生时间 |
| 类型 | 充值 / 活动扣减 / 冲红退回 / 调整 |
| 金额 | 正数入账，负数支出 |
| 变动后余额 | 操作后的余额快照 |
| 说明 | 关联的活动名称或操作备注 |

### 3.3 外挂（Hanger-on）机制

**目的**：处理非群成员随行人员的费用分摊。

- **外挂定义**：没有公积金余额的随行人员，费用由所属会员代付。
- **录入方式**：在活动录入步骤二中，每个参会会员后面可输入外挂人数（默认 0）。
- **费用计算**：外挂与正式会员同等分摊，费用全部计入所属会员账单。
- **示例**：张三带 1 个外挂，总人均 50 元 → 张三应付 50 × 2 = 100 元。

### 3.4 车辆管理

**目的**：登记群组车辆信息，用于活动车费分摊计算。

#### 3.4.1 车辆列表

| 展示字段 | 说明 |
|---------|------|
| 车牌号 | 如"粤B·12345" |
| 车型 | 如"丰田RAV4" |
| 车主 | 绑定的会员姓名 |

#### 3.4.2 车辆操作

| 操作 | 说明 |
|------|------|
| 新增车辆 | 输入车牌（必填）、车型（选填）、绑定车主（从会员列表选择） |
| 编辑车辆 | 修改信息 |
| 删除车辆 | 无关联活动时可删除 |

### 3.5 路线库管理

**目的**：预录入常走路线信息，活动创建时可直接关联。

#### 3.5.1 路线信息

| 字段 | 类型 | 说明 |
|------|------|------|
| 路线名称 | 文本 | 必填，如"梧桐山大梧桐" |
| 特点描述 | 文本 | 路线描述 |
| 海拔 | 数字（米） | 最高海拔 |
| 爬升高度 | 数字（米） | 累计爬升 |
| 路程公里数 | 数字（km） | 徒步距离 |
| 开车距离 | 数字（km） | 行车距离 |
| 预计开车时长 | 数字（分钟） | 单程车程 |
| 途经点 | 坐标数组 | 用于地图展示（起点、终点、途经点） |

#### 3.5.2 路线操作

- 新增路线、编辑路线、删除路线。
- 支持按名称搜索。

### 3.6 活动录入与结算（核心功能）

**目的**：通过 5 步向导录入活动信息，系统自动计算并执行 AA 结算。

#### 3.6.1 入口

首页醒目的 **"+ 记一笔"** 大按钮，点击进入 5 步活动向导。

#### 3.6.2 步骤一：基本信息

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 活动名称 | 文本 | ✓ | 如"4.15 梧桐山登山" |
| 活动时间 | 日期 | ✓ | 日期选择器 |
| 活动地点 | 文本 | ✗ | 如"深圳梧桐山" |
| 关联路线 | 选择 | ✗ | 从路线库选择，自动带出海拔、距离等信息 |
| 备注 | 文本 | ✗ | |

#### 3.6.3 步骤二：参与人员与外挂

- 展示当前群组全部会员，勾选参与本次活动的会员。
- 每个已勾选会员右侧显示外挂人数输入框（NInputNumber，默认 0）。
- 支持搜索会员名称快速筛选。
- **缺席标记**：已报名但未参加的会员可标记为"缺席"。

#### 3.6.4 步骤三：车辆与分摊模式

**分摊模式选择**（单选）：

| 模式 | 说明 |
|------|------|
| **全员平摊** | 所有车费由全体参与人（含外挂）平均分摊 |
| **按车分摊** | 每辆车的费用由该车乘客分摊 |

**车辆录入**：
- 从车辆库选择本次出行的车辆。
- 为每辆车勾选乘客（从已选参与人中选择）。
- 选择"按车分摊"模式时，每辆车的费用独立计算。

#### 3.6.5 步骤四：费用录入

动态添加费用条目，每条包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 费用类别 | 选择 | ✓ | 食材费 / 油费 / 路费 / 停车费 / 住宿费 / 其他 |
| 金额 | 数字 | ✓ | 精确到分 |
| 垫付人 | 选择 | ✓ | 从参与人中选择 |
| 是否车辆费用 | 开关 | ✗ | 标记为车辆相关费用（用于按车分摊） |
| 关联车辆 | 选择 | 条件 | 按车分摊模式下，车辆费用需关联具体车辆 |
| 备注 | 文本 | ✗ | |

#### 3.6.6 步骤五：智能结算

系统自动完成以下计算并展示结算单：

1. **计算人均费用**：
   - 总人头 = 会员数 + 全部外挂数
   - 公共费用人均 = 公共费用总额 ÷ 总人头
   - 车辆费用人均 = 按选定模式计算（全员平摊或按车分摊）

2. **生成结算单**（表格展示）：

| 姓名 | 人头数 | 应付金额 | 垫付金额 | 净结算 | 余额状态 |
|------|:------:|-------:|-------:|------:|---------|
| 张三 | 2 | 200.00 | 300.00 | -100.00 | 需报销 ¥100 |
| 李四 | 1 | 100.00 | 0.00 | 100.00 | 余额充足，扣减 |
| 王五 | 1 | 100.00 | 0.00 | 100.00 | 余额不足，需补 ¥30 |

3. **结算执行**：
   - 管理员确认后点击"确认结算"。
   - 系统在一个原子事务中完成：扣减余额 → 写入流水 → 更新活动状态。

### 3.7 混合结算与资金调节

**目的**：处理垫付报销和余额不足的场景。

#### 3.7.1 报销模式（管理员选择）

| 模式 | 说明 | 资金池影响 |
|------|------|----------|
| **走公款** | 系统增加垫付人的余额（等于报销金额） | 消耗资金池 |
| **走现金** | 系统提示管理员线下转账给垫付人 | 不消耗资金池 |

- 每次活动可选择报销模式，默认"走公款"。

#### 3.7.2 余额不足处理

- 结算时若会员余额不足以支付应付金额，系统扣减可用余额至 0，剩余部分标记为"需补现金"。
- 管理员线下收款后，使用"一键收款充值"完成：系统充值 → 自动扣减欠款。

#### 3.7.3 资金调节表

活动结算后自动生成，展示：
- 每位参与者的余额变动明细
- 需补现金的人员名单和金额
- 需报销的垫付人和金额
- 资金池总余额变化

### 3.8 活动编辑与冲红重算

**目的**：支持已结算活动的修改，自动处理资金回滚和重新结算。

#### 3.8.1 编辑入口

活动列表页 → 已结算活动 → 点击"编辑"按钮（铅笔图标）。

#### 3.8.2 编辑流程

1. 弹出活动向导（与创建相同的 5 步），回显原始数据，顶部显示"编辑模式"提示条。
2. 管理员修改所需内容（金额、人员、外挂数量等）。
3. 点击保存时系统自动执行：

| 步骤 | 操作 | 说明 |
|:----:|------|------|
| 1 | **冲红** | 将原活动所有扣减金额全额加回会员余额，写入 `reversal` 类型流水 |
| 2 | **重算** | 根据修改后的数据重新运行结算算法 |
| 3 | **入账** | 按新结算结果扣减余额，写入新的 `debit` 类型流水 |

全部在一个数据库事务中执行，保证原子性。

#### 3.8.3 差异确认

- 保存前弹出差异确认框，展示：
  - 人均费用变化（旧 → 新）
  - 每位会员的余额调整明细（如"张三减少 10 元，李四减少 10 元"）
- 若已线下收款，提醒管理员根据差额补收或退还。

#### 3.8.4 审计日志

每次修改自动记录：
- 修改时间
- 修改内容（字段级别差异，如"油费 200 → 250"）
- 操作人（当前登录的管理员）

### 3.9 地图与可视化

**目的**：在地图上展示活动路线和参与人信息。

- **地图服务**：高德地图 JS API。
- **活动地图**：
  - 展示路线的起点、终点、途经点标记。
  - 点击标记可查看活动名称、日期、参与人数。
- **参与者展示**：展示参加人员头像及参加时间。

### 3.10 报表与统计

#### 3.10.1 单次活动结算单

| 展示项 | 说明 |
|-------|------|
| 活动基本信息 | 名称、日期、地点、路线 |
| 费用明细 | 分类展示各项费用和垫付人 |
| 参与人结算表 | 姓名、人头数、应付、垫付、净结算 |
| 车辆分摊明细 | 各车辆费用和分摊方式 |

#### 3.10.2 全员资金总表

| 字段 | 说明 |
|------|------|
| 姓名 | 会员名称 |
| 实时余额 | 当前公积金余额 |
| 累计充值 | 历史充值总额 |
| 累计消费 | 历史扣减总额 |
| 活动次数 | 参与活动总次数 |

- 支持按余额、消费排序。
- 底部展示资金池汇总（总余额、总充值、总消费）。

#### 3.10.3 个人流水

## 4. 业务规则与算法

### 4.1 外挂算法

```
单价 = 总费用 ÷ (会员数 + 外挂总数)
会员A（带N个外挂）应付 = 单价 × (1 + N)
```

**示例**：总费用 500 元，3 个会员 + 2 个外挂（张三带 1 个，李四带 1 个）

- 单价 = 500 ÷ 5 = 100 元
- 张三应付 = 100 × 2 = 200 元
- 李四应付 = 100 × 2 = 200 元
- 王五应付 = 100 × 1 = 100 元

### 4.2 车费算法

#### 模式 A：全员平摊

```
车费人均 = 总车费 ÷ 总人头数（含外挂）
```

#### 模式 B：按车分摊

```
某车费用人均 = 该车费用 ÷ 该车实际乘坐人头数（含外挂）
```

**示例**：A 车油费 200 元，坐了 4 人头；B 车油费 150 元，坐了 3 人头

- A 车人均 = 200 ÷ 4 = 50 元
- B 车人均 = 150 ÷ 3 = 50 元

### 4.3 缺席惩罚

报名但未参加的会员（标记为"缺席"）：
- 默认承担车费和公摊损耗。
- 不退还已扣减的预充值。
- 管理员可手动调整缺席者的分摊比例。

### 4.4 结算优先级

1. 尝试从余额扣减应付金额。
2. 余额充足 → 直接扣减，状态标记"已扣减"。
3. 余额不足 → 余额扣减至 0，差额标记"需补现金 ¥X"。
4. 有垫付金额 → 根据报销模式处理（走公款增余额 / 走现金提示线下）。

### 4.5 冲红重算规则

1. **冲红**：将原结算产生的所有 `debit` 流水逐条反向操作，每笔生成对应 `reversal` 流水。
2. **重算**：以修改后的数据重新执行结算算法。
3. **入账**：按新结果写入 `debit` 流水并扣减余额。
4. 三个步骤在一个数据库事务中完成，任一步失败全部回滚。

## 5. 用户流程

### 5.1 管理员核心流程：创建活动并结算

```
首页 → 点击"+ 记一笔"
  → 步骤一：填写活动名称、日期、地点，选择路线
  → 步骤二：勾选参与会员，输入外挂人数
  → 步骤三：选择分摊模式，选车辆，分配乘客
  → 步骤四：逐条录入费用（类别、金额、垫付人）
  → 步骤五：查看结算预览 → 确认结算
  → 系统执行扣减 → 生成结算单
```

### 5.2 活动编辑冲红流程

```
活动列表 → 选择已结算活动 → 点击编辑
  → 加载原始数据到向导 → 修改内容
  → 点击保存 → 弹出差异确认框
  → 确认 → 系统执行冲红+重算+入账
  → 更新结算单
```

### 5.3 手动充值流程

```
会员列表 → 选择会员 → 点击充值
  → 输入金额 → 确认
  → 系统增加余额 → 写入 topup 流水
```

### 5.4 一键收款充值流程

```
会员列表 → 点击"一键收款充值"
  → 系统展示所有欠款会员列表
  → 管理员勾选已收款的会员 → 输入各人收款金额
  → 确认 → 系统批量充值并补扣欠款
```

### 5.5 普通会员查看流程

```
登录（输入会员密码）
  → 首页展示活动历史列表
  → 点击活动 → 查看结算单详情
```

## 6. 非功能性需求

### 6.1 终端适配

- 移动端优先设计（H5 Web 应用）。
- 适配 iOS Safari 和 Android Chrome。
- 触屏友好（按钮最小 44×44px 可点击区域）。
- 支持 PC 浏览器访问（自适应布局）。

### 6.2 数据安全

- 资金流水表（transactions）不可修改、不可删除，仅追加写入。
- 所有金额操作在数据库事务中执行。
- 审计日志记录每一次数据变更。
- 群组密码 bcrypt 加密存储。

### 6.3 可用性

- 首页加载时间 < 2 秒（3G 网络）。
- 结算计算响应 < 500ms。
- 长列表使用虚拟滚动，支持 1000+ 条流水记录。

### 6.4 部署

- 部署目标：阿里云 ECS 单机部署。
- 数据库：SQLite 单文件，支持每日自动备份。
- 架构支持后续迁移至 MySQL。

## 7. 术语表

| 术语 | 定义 |
|------|------|
| 公积金 | 群成员预充值的活动经费余额 |
